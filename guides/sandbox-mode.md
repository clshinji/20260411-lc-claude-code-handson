# Claude Code のサンドボックスモード

コマンド実行を安全に隔離する「サンドボックス」機能の使い方です。

---

## サンドボックスとは？

Claude Code がBashコマンドを実行する際に、**OS レベルでファイルシステムとネットワークを隔離**する機能です。

- 意図しないファイルの書き換えを防止
- 悪意のある依存関係やスクリプトの影響を制限
- プロンプトインジェクション攻撃への対策

---

## 2つの権限モード

### Auto-allow モード
サンドボックス**内**のコマンドは許可プロンプトなしに自動実行されます。

- 許可プロンプトが少なく、作業がスムーズ
- サンドボックス外へのアクセスは通常の許可フローに戻る
- 明示的な deny ルールは常に尊重される

### Regular permissions モード
すべてのコマンドが標準的な許可フローを経由します。

- サンドボックス内のコマンドも明示的な許可が必要
- より厳密な制御が可能
- 許可プロンプトが増える

---

## 隔離の範囲

### ファイルシステム

| 操作 | デフォルトの範囲 |
|---|---|
| 書き込み | 現在のワーキングディレクトリ以下のみ |
| 読み取り | コンピュータ全体（一部ブロックを除く） |

ワーキングディレクトリ外への書き込みは、明示的に許可する必要があります。

### ネットワーク

- 承認済みドメインのみアクセス可能
- 未承認ドメインへのアクセスは許可プロンプトが発生
- プロキシ経由で全ネットワークアクセスを監視

---

## 有効化する

```bash
/sandbox
```

コマンドを入力するとメニューが開き、モードを選択できます。

---

## 詳細設定（`.claude/settings.json`）

書き込みや読み取りを許可するパスを細かく指定できます。

```json
{
  "sandbox": {
    "enabled": true,
    "filesystem": {
      "allowWrite": ["~/.kube", "/tmp/build"],
      "denyRead": ["~/"],
      "allowRead": ["."]
    }
  }
}
```

### パスの書き方

| プレフィックス | 意味 | 例 |
|---|---|---|
| `/` | ファイルシステムルートからの絶対パス | `/tmp/build` |
| `~/` | ホームディレクトリからの相対パス | `~/.kube` |
| `./` またはなし | プロジェクトルートからの相対パス | `./output` |

---

## プラットフォーム別の対応状況

| OS | 使用技術 | 追加インストール |
|---|---|---|
| macOS | Seatbelt | 不要 |
| Linux / WSL2 | bubblewrap + socat | 必要 |
| WSL1 | - | 非対応 |

### Linux / WSL2 の場合

```bash
# Ubuntu / Debian
sudo apt-get install bubblewrap socat

# Fedora
sudo dnf install bubblewrap socat
```

---

## 注意点

- `watchman` と `docker` はサンドボックス非対応（設定で除外可能）
- ネットワーク制限はドメインレベルのため、domain fronting などで回避される可能性がある
- 若干のパフォーマンスオーバーヘッドあり（最小限）
